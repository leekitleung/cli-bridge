# ADR-0001: v1.5 Automation Boundary

Status: Accepted

Date: 2026-06-08

Update: ADR-0002 supersedes web-dom automatic ChatGPT send as the active v1.5b
route. v1.5a automatic fill remains accepted; v1.5b now proceeds through local
review-only command transport.

## Context

CLI Bridge has intentionally operated as a safe, semi-automatic relay. The
existing README and roadmap prohibited automatic ChatGPT send, automatic agent
loops, and real Codex PTY delivery. That boundary made the browser extension a
manual clipboard helper rather than an automation endpoint.

The project direction is now B: unfreeze automation so data can move through both
ends automatically. This reverses part of the previous safety boundary and must
be recorded before implementation.

## Decision

Automation is allowed only as explicit, staged, auditable capabilities:

1. v1.5a may add a local outbound prompt queue and browser-extension polling
   that automatically fills the ChatGPT composer. It must not click send,
   simulate keyboard input, call `requestSubmit`, or bypass user submission.
2. Later v1.5 slices may add bounded automation only behind explicit opt-in
   controls, visible logs, and round limits. ADR-0002 chooses local review-only
   command transport for v1.5b instead of automatic ChatGPT Web send.
3. The following remain hard prohibited: generic `/exec`, `/shell`, `/run`, or
   `/command` endpoints; attaching to arbitrary existing terminals; stop-session
   behavior for control; reading browser cookies, localStorage, or page secrets;
   automatic commit, push, merge, or PR creation; persisting raw unredacted
   content.

## Risk Acceptance

Programmatically driving a logged-in ChatGPT Web session, especially automatic
send, may violate service expectations or trigger account risk. This risk cannot
be removed in code. v1.5a does not perform automatic send, and ADR-0002 defers
web-dom automatic send in favor of local CLI command transport.

## Consequences

- The README and roadmap must describe automation as restricted and opt-in, not
  categorically forbidden.
- Tests must distinguish v1.5a automatic fill from still-prohibited automatic
  send and shell execution.
- Every automation expansion after v1.5a needs its own implementation handoff or
  ADR update before code.
