# ADR-0002: v1.5b Command Transport Instead of Web-DOM Auto Send

Status: Accepted

Date: 2026-06-08

Update: ADR-0023 accepts a newer, narrower ChatGPT Web automation boundary.
ADR-0023 supersedes only this ADR's categorical Web-DOM auto-send rejection;
all command, shell, arbitrary argv, permission-bypass, and unbounded-loop
rejections remain active.

## Context

ADR-0001 allowed staged automation and left later slices open for automatic
ChatGPT Web send, automatic extraction, bounded loops, and real Codex PTY
delivery. Further local verification found that both target CLIs expose stable
non-interactive modes designed for scripted use:

- Codex CLI `0.135.0`: `codex exec`, `codex exec review`, stdin prompts,
  `--json`, `--output-last-message`, `--output-schema`, `--ephemeral`, and
  `-s read-only` where applicable.
- Claude Code `2.1.168`: `claude -p`, `--output-format json`,
  `--json-schema`, `--tools ""`, `--disallowed-tools`, `--permission-mode plan`,
  and `--no-session-persistence`.

This satisfies the roadmap requirement that command transport needs a stable
non-interactive review-only CLI mode and separate approval.

## Decision

v1.5b will use local command transport review-only adapters instead of web-dom
automatic ChatGPT send.

The web-dom auto-send path is superseded for v1.5b and deferred indefinitely.
v1.5a's outbound fill queue remains useful as a manual ChatGPT Web assistive
path, but it is no longer the active route for end-to-end automation.

Approved v1.5b scope:

- fixed, allowlisted command invocations only;
- argv-array process execution with `shell: false`;
- Codex and Claude review-only adapters;
- no-tools/read-only/plan constraints;
- schema/parser validation into ReviewResult;
- timeout and output-size limits;
- audit events with command metadata and no raw content persistence;
- PendingReview lifecycle integration without auto-executing follow-up prompts.

## Rejected

- automatic ChatGPT Web send;
- `requestSubmit`, `.submit()`, send-button click automation, or keyboard
  simulation;
- generic command, shell, run, or exec HTTP endpoints;
- user-provided commands or arbitrary argv;
- dangerous permission bypass flags;
- automatic source-agent feedback or unbounded loops.

## Consequences

- v1.5b risk drops from high account-risk web automation to medium local
  execution-risk automation.
- Implementation must start with a command allowlist runner and fake-runner
  tests before any real adapter code.
- Documentation and tests must keep web-dom auto-send prohibited while allowing
  fixed review-only command transport.
- ADR-0001 still governs v1.5a and future automation, but ADR-0002 is the active
  route for v1.5b.
